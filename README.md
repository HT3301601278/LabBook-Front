# 实验室预约管理系统（LabBook）

## 1. 项目概述

实验室预约管理系统（LabBook）是一款基于Vue.js开发的现代化Web应用，采用前后端分离架构，利用RESTful API实现数据交互。系统旨在提升学校实验室资源的利用效率与管理水平，实现实验室信息维护、预约管理、设备报修与检修、公告发布以及数据统计等全流程智能化管理。

## 2. 系统特点

- **多角色管理**：支持系统管理员、实验室管理员及学生三种用户角色，依据权限分配不同功能。
- **直观界面**：基于Element UI构建现代化、响应式的用户界面，适配多种设备。
- **安全认证**：采用基于Token的用户身份验证，确保系统安全性。
- **智能预约**：提供时间冲突检测与预约时长限制，确保预约流程高效有序。
- **全流程追踪**：覆盖预约申请、使用、报修、检修等环节，记录详细操作信息。
- **3D模型交互**：利用Three.js实现实验室3D模型的旋转、缩放及光照调节，直观展示实验室布局。
- **数据可视化**：运用ECharts等图表工具，直观展示用户分布、实验室状态及使用统计数据。

## 3. 功能模块划分

系统根据用户角色和使用场景分为以下主要模块：

### 3.1 系统管理员模块
- **用户管理**：管理所有角色账号（系统管理员、实验室管理员、学生），包括注册审核和权限配置。
- **实验室分类与信息管理**：创建实验室分类、维护实验室基本信息、设置开放时间、上传使用手册及3D模型文件。
- **公告管理**：发布和管理系统公告，确保信息及时传达。
- **预约与报修记录管理**：查看所有预约、报修及检修记录，进行统计与数据可视化分析。

### 3.2 实验室管理员模块
- **实验室信息管理**：维护实验室详情、管理开放时间与预约规则，上传相关文档和3D模型。
- **预约审核**：审核学生提交的预约申请，并实时更新预约状态。
- **报修处理**：接收并处理设备报修申请，跟踪检修进度与记录管理。
- **个人信息管理**：管理个人资料和账号相关信息。

### 3.3 学生模块
- **实验室浏览与预约**：浏览可用实验室、查询详细信息（位置、状态、开放时间、使用手册及3D模型），选择合适时间段提交预约申请。
- **设备报修**：遇到设备故障时，可在线提交报修申请，并跟踪处理状态。
- **个人信息管理**：查看及更新个人资料和预约记录。

## 4. 主要功能点说明

### 4.1 用户管理

#### 4.1.1 用户注册
- **功能描述**：学生通过注册页面填写基本信息（用户名、密码、姓名、学号、学院、专业、电话等）创建账号，同时支持学院与专业的级联选择。
  
#### 4.1.2 用户登录
- **功能描述**：所有用户通过统一登录入口进入系统。
- **实现细节**：
  - 支持角色选择（系统管理员、实验室管理员、学生）
  - 基于Token的身份验证，登录状态保持
  - 路由守卫保障未登录用户无法访问受保护页面

#### 4.1.3 个人信息管理
- **功能描述**：用户可查看与修改个人资料，包括基本信息和密码更新。

### 4.2 实验室管理

#### 4.2.1 实验室分类管理
- **功能描述**：系统管理员可增删改查实验室分类，管理分类信息并关联相应实验室管理员。

#### 4.2.2 实验室信息管理
- **功能描述**：管理员可创建、编辑和维护实验室基本信息。
- **实现细节**：
  - 管理实验室名称、编号、位置、所属分类等信息
  - 设置实验室开放时间、最大预约时长与使用状态（空闲、使用中、已预约）
  - 上传实验室使用手册（PDF格式）和3D模型文件

#### 4.2.3 实验室3D模型查看
- **功能描述**：提供交互式3D模型查看功能。
- **实现细节**：
  - 基于Three.js实现3D模型渲染
  - 支持模型旋转、缩放及光照调节
  - 提供全屏查看模式

### 4.3 预约管理

#### 4.3.1 实验室预约
- **功能描述**：学生可在线预约实验室。
- **实现细节**：
  - 采用卡片式展示实验室信息，并提供按分类和名称筛选功能
  - 展示详细信息（位置、使用状态、开放时间等）
  - 支持预约时间段选择，并对预约时长进行限制检查

#### 4.3.2 预约审核
- **功能描述**：实验室管理员审核学生提交的预约申请。
- **实现细节**：
  - 展示预约申请列表，支持按状态（待审核、已通过、已拒绝）筛选
  - 提供审核操作（通过或拒绝）及预约状态更新（未开始、进行中、已完成）

#### 4.3.3 预约记录管理
- **功能描述**：用户可查看和管理自己的预约记录。
- **实现细节**：
  - 学生可取消待审核的预约
  - 管理员可结束正在进行的预约
  - 采用颜色标签对预约状态进行直观展示

### 4.4 报修与检修管理

#### 4.4.1 设备报修
- **功能描述**：用户可在线提交实验室设备报修申请。
- **实现细节**：
  - 填写报修说明，选择相关实验室信息
  - 提交后可跟踪报修状态（待处理、处理中、已完成）

#### 4.4.2 报修处理
- **功能描述**：管理员负责处理报修申请。
- **实现细节**：
  - 对报修记录进行查询和筛选
  - 填写检修人、联系电话、检修部门、检修内容、处理说明及处理时间等信息
  - 更新报修状态

#### 4.4.3 检修记录管理
- **功能描述**：提供检修记录的查看与管理功能。
- **实现细节**：
  - 展示检修记录列表，支持按检修人、内容及部门筛选
  - 提供检修详情查看

### 4.5 公告管理

#### 4.5.1 公告发布与管理
- **功能描述**：系统管理员负责公告信息的发布与维护。
- **实现细节**：
  - 支持公告的增删改查及时间管理

#### 4.5.2 公告展示
- **功能描述**：在前台页面直观展示公告信息。

### 4.6 数据可视化
- **功能描述**：通过图表展示实验室使用情况及统计数据。
- **实现细节**：
  - 使用ECharts实现图表展示，包括饼图（用户分布、实验室状态）和柱状图（实验室分类统计）

## 5. 用户角色及操作流程

### 5.1 系统管理员
- **主要职责**：
  - 管理所有用户账号与权限
  - 维护实验室分类和整体实验室信息
  - 发布系统公告，管理预约及报修记录
  - 查看数据统计与可视化报表
- **操作流程**：
  1. 登录系统
  2. 创建/编辑管理员和实验室管理员账号
  3. 审核学生注册申请
  4. 管理实验室分类与信息
  5. 发布公告与监控系统数据

### 5.2 实验室管理员
- **主要职责**：
  - 负责具体实验室的信息维护及预约审核
  - 处理设备报修和检修记录
  - 管理实验室状态与开放时间
- **操作流程**：
  1. 登录系统
  2. 维护并更新实验室信息（包括上传使用手册和3D模型）
  3. 审核学生预约申请，并更新预约状态
  4. 处理设备报修及记录检修详情

### 5.3 学生用户
- **主要职责**：
  - 浏览实验室信息，提交预约和报修申请
  - 管理个人预约记录及个人资料
- **操作流程**：
  1. 注册并登录系统
  2. 浏览可用实验室（查看详细信息、使用手册或3D模型）
  3. 选择合适时间段提交预约申请
  4. 等待预约审核结果后按预约时间使用实验室
  5. 遇到设备问题时提交报修申请，并跟踪处理状态

## 6. 异常处理与边缘情况

- **用户认证异常**：
  - 登录失败或Token过期时，系统提供错误提示并引导重新登录
  - 权限不足时跳转至403错误页面
- **预约冲突处理**：
  - 系统自动检测预约时间冲突，防止重叠预约
  - 严格控制预约时长，超过规定时长的预约将被拒绝或提示修改
- **文件上传异常**：
  - 限制文件类型（如PDF、3D模型格式）和大小，上传失败时显示详细错误信息并允许重试

## 7. 技术架构

- **前端框架**：Vue.js 2.6
- **UI组件库**：Element UI 2.15
- **路由管理**：Vue Router 3.5
- **HTTP客户端**：Axios
- **3D渲染**：Three.js
- **图标库**：Font Awesome

## 8. 注意事项

1. **系统配置**：首次使用前需正确配置后端API地址。
2. **浏览器兼容性**：建议使用Chrome、Firefox等现代浏览器以获得最佳体验。
3. **系统时间**：确保设备系统时间准确，避免预约时间错误。
4. **及时响应**：管理员需及时处理预约、报修及检修申请，确保系统数据更新准确。
